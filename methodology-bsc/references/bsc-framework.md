# BSC 四维度框架详解

> 来源：Stage 2 方法论定义

## 因果链

BSC 四个维度不是并列的四个筐，而是一条因果链——下层驱动上层：

```
学习与成长（你投入了什么能力？）
    ↓
内部流程（你做出了什么产品/流程？）
    ↓
客户（客户得到了什么价值？）
    ↓
财务（你赚到了什么钱？）
```

## 财务维度——你赚了多少钱？钱花在哪里？

**通俗问法**：
- 营收、利润、现金流、应收账款——跟去年比、跟年初定的目标比，数据差异是多少？
- 咱们的业务目前是处于导入期、成长期、还是成熟期？

**生命周期匹配**：
- 成长期：看收入增长（收入增长率、新产品收入占比）
- 保持期：看获利能力（资本报酬率、EVA、毛利率）
- 成熟期：看现金回收（经营现金流量、现金周转期）

**边界**：属于财务的是收入、利润、成本、现金流、应收账款、回款率。不属于财务的是客户数、NPS——这些是驱动因素，但本身不是钱。

## 客户维度——谁在买你的东西？他们满意吗？

**通俗问法**（五个具体问题）：
1. 我们现在有多少客户？
2. 新客户的增长率是多少？
3. 老客户流失率是多少？
4. 客户愿意推荐我们吗（NPS）？
5. 客户为公司贡献了多少利润？

加上：
- 你服务的客户是什么类型？
- 相比于竞争对手，我们的优势和劣势在哪里？

**核心指标链**：客户满意度 → 客户保持率 + 客户获得率 → 市场份额 → 客户获利率

## 内部流程维度——你是怎么把东西做出来、交付出去的？

**通俗问法**：
- 从客户提需求到需求被满足，平均需要多长时间？哪个环节最花时间？
- 过去一年，咱们在研发上投入了多少？这些投入带来的新功能/新能力，产生收入了吗？大概多久能收回研发成本？
- 客户遇到问题找我们，一次就解决的比例大概是多少？哪些问题需要反复折腾？

**三个流程**：
- 创新流程（确认需求→开发新产品）：关键指标 BET 收支平衡时间
- 经营流程（接单→交付）：关键指标是从需求提出到被满足的时长
- 售后服务流程：关键指标是一次成功率

**特别注意**：BSC 强调内部流程不能只有"经营流程"，还必须有"创新流程"——未来的种子。

## 学习与成长维度——你的人、系统、文化能支撑你的野心吗？

**通俗问法**：
- 接下来一年，咱们业务上最想突破的一件事是什么？这件事需要什么能力？咱们团队里现在有这些能力吗？
- 你做关键决策的时候，最需要知道哪些信息？这些信息你目前能拿到吗？是准时的、还是滞后的？是准确的、还是大概估的？
- 咱们的考核和奖金，跟事业部的战略目标是挂钩的吗？员工提的改进建议，有没有被采纳的机制、有没有被用上的例子？

**三个来源**：员工能力（战略工作胜任率）、信息系统能力（战略信息覆盖率）、激励与协作（目标一致性 + 建议采纳率）。

## 常见误解

1. "BSC 就是把指标分成四类列出来"——四个维度是因果关系，不是并列分类
2. "客户维度不重要，因为客户是内部的"——内部客户也有选择权
3. "学习与成长就是培训次数"——培训次数是诊断指标，不是战略指标。战略指标是"培训后，能力提高了吗？"
4. "改进措施列出来就够了"——每条改进措施需要具体动作 + 时间节点 + 可验证成果
5. "差因分析把外部原因列清楚就行了"——BSC 追问的是"外部条件不变，你自己能控制的部分里，什么可以做得不一样？"
