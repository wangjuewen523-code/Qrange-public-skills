# methodology-bsc · 平衡计分卡业务复盘 Skill

> 一个可以装进任何 AI Agent 的复盘引导技能：用平衡计分卡（BSC）四维度因果链，把零散的业务数据，一步步追问成一份有因果逻辑的结构化复盘文档。

**它不是检查清单生成器，而是一套因果推理系统。**

---

## 解决什么问题

业务复盘的三个常见病：

- **分类代替归因**：指标往"财务 / 客户 / 流程 / 学习"四个筐里一放就完了，没人追问四个筐之间的因果关系
- **甩锅代替分析**：差因分析全是"行业下行""客户预算收紧"，看不到一条内部可控因素
- **口号代替措施**：改进措施是"加强管理""提升能力"，没有动作、没有时间节点、没有验证标准

这个 Skill 干的事：像一个有经验的复盘引导师坐在你对面，把你已经填的、漏掉的、没想清楚的，一轮一轮问出来——最后给你一份可以直接拿去汇报的五段式复盘文档。

## 快速开始

### 安装

这是一个标准的 Agent Skill 包（`SKILL.md` + `references/` + `examples/`），平台无关：

- **支持技能包格式的 Agent 运行时**（Claude Code、OpenClaw、Hermes、WorkBody，或任何兼容 SKILL.md 规范的个性化 Agent）：把整个 `methodology-bsc` 文件夹放进对应的 skills 目录即可，无任何外部依赖
- **不支持技能包的 Agent**：把 `SKILL.md` 全文作为系统提示词（system prompt）注入，references 和 examples 作为上下文按需提供给模型，效果等同

### 触发

让用户直接说（任意一句）：

```
帮我做业务复盘
用 BSC 框架帮我分析一下
帮我看看这个复盘还缺什么
这个复盘够不够完整
```

或者直接粘贴一段业务数据 / 复盘材料，说"帮我分析一下"。

**不需要用户懂平衡计分卡。** Skill 会说人话，不拽术语。

## 工作原理

### 四维度因果链

BSC 的四个维度不是并列的四个筐，而是一条因果链——下层驱动上层：

```
学习与成长（你投入了什么能力？）
    ↓
内部流程（你做出了什么产品/流程？）
    ↓
客户（客户得到了什么价值？）
    ↓
财务（你赚到了什么钱？）
```

大多数复盘只做了"分类"，没做"串联"。这个 Skill 的核心价值就是逼出那条因果故事：我们投入了什么能力 → 做出了什么产品/流程 → 客户得到了什么价值 → 最终赚到了什么钱。

### 两段式交互

**第一步：问材料。** 有没有 KPI 数据、差因分析、去年的复盘报告？有就直接贴；没有就给一份框架模板，让你照着去搜集（不用全部准备完，有一块算一块）。

**第二步：框架对齐 + 缺口检测。** 把你的材料对照检测矩阵，输出一张缺口地图：

```
✅ 财务维度（有数据）
⚠️ 客户维度（有投诉记录，缺复购/流失数据）
❌ 内部流程维度
❌ 学习与成长维度

深度方面：
❌ 因果串联：四个维度各自独立，没有因果故事
⚠️ 差因分析：外部归因偏多，缺内部可控因素
⚠️ 改进措施：有方向，但缺时间节点和可验证成果
```

### 有节制的追问

缺口分级处理，不是无限追问：

| 级别                        | 缺口                                                             | 策略                      |
| --------------------------- | ---------------------------------------------------------------- | ------------------------- |
| **关键缺口（K1-K5）** | 财务空白 / 有分类无因果 / 全部外部归因 / 客户空白 / 有经营无创新 | 必须追问，最多 3 轮       |
| **次要缺口（M1-M3）** | 有口号无量化 / 措施空泛 / 维度不完整                             | 追问 1-2 轮，不配合就标注 |

**退出条件**：你明确说"没有数据""就这些了"，Skill 不纠缠，用 `[信息缺口: xxx]` 或 `[用户判断: xxx]` 标注后继续推进。

全程守住三条底线：

1. **说人话**——不说"成果指标和动因指标是否混合"，说"你这些指标里，哪些是已经发生的结果，哪些是驱动结果的原因？"
2. **给镜子不给结论**——不说"这个决策错了"，问"这个成功换一个人能做到吗？"
3. **不编造数据**——缺口显式标注，绝不用 AI 推断填充

### 输出：五段式复盘文档

```
1.1 指标回顾   —— BSC 四维度表格，缺失处显式标注
1.2 差因分析   —— 区分外部 [不可控] 与内部可控因素
1.3 重大得失   —— 得：可复现性判断 / 失：可避免性判断
1.4 改进措施   —— 具体动作 + 时间节点 + 可验证成果
1.5 因果串联   —— 四个维度串成一条因果故事，附"下次怎么做"的建议
```

## 实际效果

用一个完全不懂 BSC 的咖啡连锁区域经理（12 家门店，H1 营收下滑 40 万）做了两轮角色扮演实测：

**配合型用户**——Skill 引导他自己挖出了完整因果链：

> 考核只指营收 + 无品质专岗（学习与成长）→ 配方表无执行机制、机器无校准（内部流程）→ 出品不稳定 → 投诉 30+、复购降至约 20%（客户）→ 营收 -40 万（财务）

用户原话："这么一说出来，好像也没那么难？之前就是一直没人逼我把'想突破'翻译成'几月干什么、怎么算成'。"

**不配合型用户**（说不清、不理解、想让 AI 替他干活）——Skill 守住底线：不编造、不纠缠、标注退出，仍然产出一份带 `[信息缺口]` 标注的可用文档。

完整对话示范见 [examples/sample-conversation.md](examples/sample-conversation.md)。

## 边界（它不做什么）

- **不替你制定战略**：只做框架检测和因果追问，不给"你应该选哪个市场"的建议
- **不替你编造数据**：缺口用 `[信息缺口]` 标注，不做 AI 推断填充
- **不评价决策对错**：给镜子（"当时如果多一个信息，结果会不会不同？"），不给结论（"这个决策错了"）
- **不是完整的 BSC 实施项目**：这是轻量复盘工具，不要求你按 BSC 十步法重建管理体系
- **不覆盖市场洞察和战略规划**：宏观分析、战略议题请使用对应工具
- **因果链是理论框架，不是科学定律**：Norreklit（2000/2003）指出 BSC 因果链从未被严格实证验证——这里用它帮你把假设显式化，用你自己的业务数据去验证
