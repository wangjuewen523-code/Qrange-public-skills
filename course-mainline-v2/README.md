# course-mainline v0.2.0

将课程、书籍、分享等任意结构化材料整理成标准化 9 模块主线 Markdown 文档。

## 这是什么

你有一门课（或一本书、一场分享），想把它变成一份结构化的方法论文档——说清楚它是干什么的、核心方法是什么、怎么操作、怎么判断好坏、有什么坑。这个 Skill 帮你做这件事。

不管材料是什么形式——课堂逐字稿、Get笔记、书籍原文、PPT 文本、论文——只要用户提供了，就能用。

## 怎么用

1. 给 AI 发送材料，说「提取主线」「整理课程主线」「提取这本书的方法」「萃取这本书」等
2. AI 会先用五步流程处理，最后输出一份 9 模块的 Markdown 文档

## 五步流程

| 步骤               | 做什么                                          |
| ------------------ | ----------------------------------------------- |
| **前置检查** | 盘点材料 + 估算体积（超过安全线会提示降级方案） |
| **Step 1**   | 读取资料：结构扫描 → 深度阅读 → 外部来源补充  |
| **Step 2**   | 按模板写 9 模块产出                             |
| **Step 3**   | 脚本自动检查 + 四要素人工内容审查               |
| **Step 4**   | 复盘模板，判断是否需要调整                      |
| **Step 5**   | 向用户汇报模板健康度                            |

## 9 模块结构

```
1. 定位          — 这门课/这本书到底在教什么
2. 核心问题      — 不学它会犯什么错
3. 核心方法      — 方法论骨架长什么样
4. 操作流程      — 学完之后第一步做什么、第二步做什么
5. 判断标准      — 怎么判断自己做对了还是做错了（三列表格）
6. 适用边界      — 什么场景用它，什么场景别用
7. 常见误区      — 新人最容易踩的坑（每条配原文原话纠正）
8. 讲师观点      — 标注类型：核心主张/方法论框架/经验主张/趋势判断/引用观点
9. 资料来源      — 原材料是什么，各自贡献了什么
```

## 文件结构

```
course-mainline-v2/
├── README.md                              ← 你正在读的文件
├── SKILL.md                               ← Skill 主文件（AI 读取的指令）
├── references/
│   ├── 课程主线-产出模板.md                ← 9 模块模板（含硬约束、格式规则、展开方式建议）
│   └── checks.py                          ← 自动化检查脚本（bold 平衡、标题层级、模块数等）
└── examples/
    ├── 示例1-书籍-方法论型.md              ← 卡普兰、诺顿《平衡计分卡》
    └── 示例2-专项课-框架型.md              ← 杜威《思维的本质》
```

### 各文件说明

| 文件                                | 角色                                                                           | AI 何时读            |
| ----------------------------------- | ------------------------------------------------------------------------------ | -------------------- |
| `SKILL.md`                        | 核心指令：告诉 AI 这个 Skill 怎么用、五步流程每一步做什么、硬约束是什么        | Skill 触发时自动加载 |
| `references/课程主线-产出模板.md` | 9 模块模板：定义每个模块要回答的问题、frontmatter 格式、格式规则、展开方式建议 | AI 写产出时参考      |
| `references/checks.py`            | 质量脚本：检查 bold 配对、标题层级、粗体标点、模块数量、Section 5 存在性       | AI 写完产出后执行    |
| `examples/示例1-*`                | 书籍 + 方法论型示例：因果链框架 + 操作流程 + 书中多案例的组织方式              | 用户想看参考时读取   |
| `examples/示例2-*` | 框架型理论著作示例：五步法 + 三层思维 + 错误来源 + 训练原则 | 用户想看参考时读取 |

## 安装

将 `.skill` 文件放到对应环境的 skills 目录：

- **Claude Code 插件**：`<plugin>/skills/course-mainline-v2/`
- **OpenCloud / Hermes**：按平台规范部署
